# Tools I Lean On

## Development and AI

**Claude Code** — My primary coding environment. I use it for everything from architecture decisions to debugging to building features end-to-end. PowSignal is almost entirely Claude Code.

**Cursor** — AI-native code editor. Use it alongside Claude Code depending on the task. The inline editing and codebase context are genuinely useful.

**Supabase** — Postgres database with a great developer experience. Use it on PowSignal and recommend it for any product that needs a real database without the infrastructure overhead.

**Vercel** — Deployment and hosting. Fast, zero-config for Next.js projects, and the preview deployment workflow is excellent for getting feedback quickly.

**Lovable** — Good for rapid UI prototyping. Useful when you want to get something visual in front of someone fast without writing a lot of CSS.

## Product and Analytics

**Amplitude** — Product analytics. Good for funnel analysis, retention, and behavioral cohorts. I've used it to build attribution frameworks and set up event tracking from scratch.

**Notion** — Project management and documentation. I use it for roadmaps, PRDs, and async communication with stakeholders.

**Figma** — Design collaboration. I'm not a designer but I can work in Figma, create wireframes, and give useful feedback on mocks.

**Postman** — API testing and documentation. Useful for understanding integrations and debugging third-party APIs.

## Automation

**Zapier** — Workflow automation for non-technical teams. Good for connecting tools that don't have native integrations.

**Rippling** — HR and payroll automation. Used it at TCP to build the contractor payroll automation that saved $100K annually.

## Communication

**Linear** — Issue tracking when you need something lighter than JIRA.

**JIRA** — For teams that need more structure. I've used it at scale with cross-functional teams.
